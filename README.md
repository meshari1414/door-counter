# عدّاد الداخلين (Door Counter PWA)

تطبيق ويب تقدمي (PWA) يعدّ الأشخاص الداخلين والخارجين من الباب باستخدام كاميرا الجهاز ورؤية حاسوبية تعمل **محلياً بالكامل** على الجهاز.

🌐 **المعاينة:** https://meshari1414.github.io/door-counter/

## ✨ المميزات

- 🎯 **كشف ذكي بالذكاء الاصطناعي** — TensorFlow.js + COCO-SSD
- ✋ **خط افتراضي قابل للسحب بالإصبع** على الفيديو مباشرة
- 🚦 **حد سعة قصوى** مع تنبيه بصري + اهتزاز + صوت
- 📊 **رسم بياني للساعات** يظهر أوقات الازدحام
- 💓 **اهتزاز خفيف** عند كل عملية عد (Haptic Feedback)
- 📱 **مشاركة الجلسة بين جوالين** — جوال على الباب وجوال آخر يشاهد لايف
- 📄 **تصدير PDF** مع رسم بياني وجدول كامل
- 📊 **تصدير CSV** للأحداث
- 🔌 **يعمل دون إنترنت** بعد التحميل الأول
- 📲 **قابل للتثبيت** على الشاشة الرئيسية كتطبيق
- 🌐 **واجهة عربية** (RTL) تماماً

## 🚀 التشغيل المحلي

التطبيق يحتاج **HTTPS** أو `localhost` لاستخدام الكاميرا.

```bash
cd door-counter
python -m http.server 8080
```
ثم افتح: `http://localhost:8080`

## 📱 المعاينة على الآيفون (الأسهل)

عبر **GitHub Pages** — مجاني وفوري:

1. ادخل: https://github.com/meshari1414/door-counter/settings/pages
2. **Source**: `Deploy from a branch`
3. **Branch**: `main` / `(root)` → **Save**
4. انتظر دقيقة → افتح: `https://meshari1414.github.io/door-counter/`
5. على Safari في الآيفون → زر المشاركة → **"إضافة إلى الشاشة الرئيسية"**

## ⚙️ نصائح للحصول على عدّ دقيق

- ضع الآيفون **مرتفعاً** بزاوية تطلّ على الباب (مثلاً 2-3 أمتار)
- استخدم خطاً **أفقياً** إذا كانت الكاميرا تطلّ من الأعلى
- اسحب الخط بالإصبع لتحديد موقع العبور بدقة
- الإضاءة الجيدة = دقة أعلى بكثير
- ارفع/اخفض حد الثقة من الإعدادات حسب الإضاءة

## 📡 إعداد Firebase للمشاركة بين الجوالات

ميزة "مشاركة الجلسة" تحتاج Firebase Realtime Database (مجاني):

### 1. أنشئ مشروع Firebase
- ادخل: https://console.firebase.google.com
- **إضافة مشروع** → اختر اسم
- ادخل **Realtime Database** → **Create database** → **Start in test mode** (أو رتّب القواعد)

### 2. اربط تطبيق ويب
- في صفحة المشروع: أيقونة **`</>`** (Add app → Web)
- انسخ كائن `firebaseConfig`

### 3. ضع الإعدادات في المشروع
- انسخ `firebase-config.example.js` إلى `firebase-config.js`
- استبدل القيم بقيم مشروعك:

```js
window.FIREBASE_CONFIG = {
  apiKey: "...",
  authDomain: "...",
  databaseURL: "...",
  projectId: "...",
  // ...
};
```

### 4. قواعد الأمان (اختياري)
في **Realtime Database → Rules** ضع:
```json
{
  "rules": {
    "rooms": {
      "$code": {
        ".read": true,
        ".write": true
      }
    }
  }
}
```
> ⚠️ هذه قواعد مفتوحة للتجربة. للإنتاج: أضف انتهاء صلاحية تلقائي وقواعد كتابة محدودة.

### 5. ارفع `firebase-config.js`
ملاحظة: الملف موجود في `.gitignore` لحماية مفاتيحك. ارفعه يدوياً للسيرفر أو أضفه كـ GitHub secret.

## 🔄 كيف تعمل المشاركة بين الجوالات

```
[الجوال 1: على الباب]                    [الجوال 2: لوحة التحكم]
        │                                          │
        ├─ يكشف الأشخاص                         │
        ├─ يعدّ الدخول/الخروج                  │
        ├─ يكتب لـ Firebase ───────────────────┤
        │                                          ├─ يقرأ من Firebase لايف
        │                                          ├─ يعرض الأرقام والرسم البياني
        │                                          └─ يستقبل تنبيه السعة
```

**كيفية الاستخدام:**
1. على الجوال 1: افتح التطبيق → اضغط **مشاركة** → **إنشاء جلسة الآن**
2. سيظهر QR + كود + رابط
3. على الجوال 2: امسح QR (أو افتح الرابط، أو ادخل الكود يدوياً)
4. ستظهر الأرقام لايف على الجوال 2

## 🧠 كيف يعمل التطبيق؟

1. الكاميرا تلتقط فيديو حياً
2. كل ~100ms يمر إطار على نموذج COCO-SSD لكشف الأشخاص
3. كل شخص مكتشف يُربط بـ "track" مستمر بناءً على IoU + المسافة
4. عند عبور مركز الـ track للخط الافتراضي، يُحدد الاتجاه:
   - عبور في اتجاه الدخول → +1 دخول
   - عبور في الاتجاه المعاكس → +1 خروج
5. كل track يُعدّ مرة واحدة فقط لمنع التكرار
6. السجل يُحفظ محلياً ويُمكن تصديره PDF/CSV

## 🔒 الخصوصية

- ✅ كل المعالجة على الجهاز
- ✅ لا يُرسل الفيديو أو الصور لأي مكان
- ✅ البيانات تُحفظ محلياً (`localStorage`)
- ⚠️ عند تفعيل المشاركة: **الأرقام فقط** ترسل لـ Firebase (بدون فيديو)
