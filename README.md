# عدّاد الداخلين (Door Counter PWA)

تطبيق ويب تقدمي (PWA) يعدّ الأشخاص الداخلين والخارجين من الباب باستخدام كاميرا الجهاز ورؤية حاسوبية تعمل **محلياً بالكامل** على الجهاز (لا يُرسل أي فيديو لأي خادم).

## ✨ المميزات

- كشف الأشخاص بالذكاء الاصطناعي (TensorFlow.js + COCO-SSD)
- خط افتراضي قابل للتعديل (أفقي/عمودي + موضعه)
- تتبع الأشخاص بين الإطارات لمنع العد المزدوج
- إحصائيات: داخل / خارج / داخل الآن
- سجل الأحداث + تصدير CSV
- يعمل دون إنترنت (Service Worker)
- يمكن إضافته للشاشة الرئيسية كتطبيق
- واجهة عربية (RTL)

## 🚀 التشغيل المحلي

التطبيق يحتاج **HTTPS** أو `localhost` لاستخدام الكاميرا.

### من ويندوز عبر Python:
```bash
cd door-counter
python -m http.server 8080
```
ثم افتح: `http://localhost:8080`

### من ويندوز عبر Node:
```bash
npx serve door-counter
```

## 📱 المعاينة على الآيفون

السبيل الأسرع: نشر على **GitHub Pages**

1. ارفع المجلد لمستودع GitHub
2. Settings → Pages → فعّل النشر من `main` branch
3. ستحصل على رابط `https://USER.github.io/REPO/door-counter/`
4. افتح الرابط على Safari في الآيفون
5. اضغط زر المشاركة → "إضافة إلى الشاشة الرئيسية"

## ⚙️ نصائح للحصول على عدّ دقيق

- ضع الآيفون **مرتفعاً** بزاوية تطلّ على الباب (مثلاً 2-3 أمتار من الأرض)
- استخدم خطاً **أفقياً** إذا كانت الكاميرا تطلّ من الأعلى
- اختر اتجاه الدخول الصحيح من الإعدادات
- ارفع حد الثقة في الإضاءة الجيدة، واخفضه في الإضاءة الضعيفة
- الإضاءة الجيدة تحسّن الكشف بشكل كبير

## 🧠 كيف يعمل؟

1. الكاميرا تلتقط فيديو حياً
2. كل ~100ms يمر إطار على نموذج COCO-SSD لكشف الأشخاص
3. كل شخص مكتشف يُربط بـ "track" مستمر بناءً على IoU + المسافة
4. عند عبور مركز الـ track للخط الافتراضي، يُحدد الاتجاه:
   - عبور في اتجاه الدخول → +1 دخول
   - عبور في الاتجاه المعاكس → +1 خروج
5. كل track يُعدّ مرة واحدة فقط لمنع التكرار

## 🔒 الخصوصية

- كل المعالجة تتم على الجهاز
- لا يُرسل الفيديو أو الصور لأي مكان
- البيانات تُحفظ محلياً في `localStorage`
